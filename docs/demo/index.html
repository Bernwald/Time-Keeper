Drei Richtungen für die Kunden-Oberfläche

Wie soll die App den Kunden anfühlen?

Drei Mockups mit Mock-Daten. Daten-Orchestrator läuft im Hintergrund — der Kunde sieht nur seine digitalen Mitarbeiter und ein Dashboard, das den Nutzen sichtbar macht. Klick eine Karte an und blättere durch.

Konzept A →

Agent-Studio

Chat als Heimat — Agenten als Werkzeuge

Wie ChatGPT, aber jeder Eintrag in der Sidebar ist ein digitaler Mitarbeiter. Eingaben per Text, Stimme oder Transkription. KPIs als Welcome-Screen.

  • ✓ Sidebar mit Agenten + Konversationen
  • ✓ Modell-Switch & Voice-Input
  • ✓ Agent-Briefing als Side-Panel
Konzept B →

Cockpit

Chat links, Live-Übersicht rechts

Geteilter Bildschirm: Anwenden links, Beobachten rechts. Manager sieht KPIs immer im Blick, ohne Tab-Hopping. Auf Mobile gestapelt.

  • ✓ Chat permanent sichtbar
  • ✓ Tabs: Übersicht · Telefon-Live · Prozesse
  • ✓ Eine Seite — alles auf einen Blick
Konzept C →

Digitale Belegschaft

Galerie deiner KI-Mitarbeiter

Landing wie ein Org-Chart: jede Agent-Karte ist ein Kollege mit Rolle, Status, KPIs. Klassischer CEO-Blick, klare Hierarchie. Skaliert für 20+ Agenten.

  • ✓ KPI-Hero + Agent-Galerie
  • ✓ Status & Mini-KPIs pro Agent
  • ✓ Activity-Feed über alle Agenten
Hinweis für die Vorführung: Alle Daten in den Demos sind beispielhaft. Ziel ist, den Look & Feel zu zeigen — nicht Funktionalität. Der Daten-Orchestrator (Sources, Embeddings, Pipeline) bleibt absichtlich unsichtbar; der Berater pflegt ihn im Admin-Bereich.
